# Contributing

Thanks for taking the time to contribute. Here’s how to jump in without confusion.

## How to propose changes
1. Fork the repo.
2. Create a new branch for your change.
3. Make your updates with clear, focused commits.
4. Test your work before opening a pull request.
5. Submit a PR describing what you changed and why.

## Code style
- Use meaningful variable and function names.
- Keep functions small and clear.
- Follow the existing formatting patterns.
- For React, keep components simple and avoid unnecessary state.

## Commit messages
Keep them short and clear:
- feat: add new feature  
- fix: resolve bug  
- docs: update documentation  
- refactor: improve structure without changing behavior

## Pull requests
A good PR has:
- A clear summary of the change
- Only one main change per PR
- Screenshots for UI updates
- Tests when applicable

## Reporting bugs
Open an issue with:
- What happened  
- What you expected  
- Steps to reproduce  
- Screenshots or logs if helpful

## Feature requests
Explain:
- The problem  
- Why it matters  
- How you imagine the solution working

## Conduct
By participating, you agree to follow the project’s Code of Conduct.
